# BoxX 首页与侧边栏重设计

> 日期：2026-04-04
> 目标：提升 BoxX 的产品化和专业化水平，参考 Surge Activity 页面风格

## 设计原则

1. **性能第一** — 不为视觉效果牺牲性能
2. **窗口关闭 = 零消耗** — 主窗口关闭时，停止一切 UI 侧轮询和数据采集，释放内存
3. **原生组件优先** — 全部使用 SwiftUI 原生组件，不使用 Canvas 或自定义绘制
4. **信息层级清晰** — 大数字 + 小标签的仪表盘风格，主次分明

## 一、侧边栏重设计

### 1.1 当前结构（9 项，4 组）

```
（通用）概览、策略组
（规则）路由规则、规则集、内置规则
（监控）请求、日志
（管理）订阅、设置
```

### 1.2 新结构（9 项，重新分组）

```
概览                          ← 无分组标题，默认选中

代理 ─────────────────────
  策略组
  订阅

规则 ─────────────────────
  路由规则
  规则集
  DNS

          (Spacer)

请求                          ← 底部区域，无分组标题
日志
设置
```

### 1.3 变更说明

| 变更 | 原因 |
|------|------|
| 订阅从"管理"移到"代理" | 订阅是代理节点的来源，和策略组放一起符合用户心智 |
| 新增 DNS 入口，归入"规则" | DNS 配置是路由决策的一部分，sing-box 用户高频操作 |
| 内置规则合并到"规则集"内部 | 内置规则是只读参考数据，不值得独占侧边栏入口；在规则集页面用标签页切换（自定义 / 内置） |
| 请求、日志、设置推到底部 | 辅助功能，用 Spacer 推到底部，主次分明 |
| 移除"监控"和"管理"分组标题 | 底部项无需分组标题，视觉更简洁 |

### 1.4 实现要点

- 使用 `List` + `Section` 构建上半部分（概览、代理、规则）
- 底部区域用 `Spacer()` 推到列表底部
- 底部三项之间可选用分割线或直接紧排，不加分组标题
- `SidebarTab` 枚举更新：移除 `builtinRules`，新增 `dns`

## 二、概览页重设计

### 2.1 顶部状态条

单行横排，展示当前核心状态信息：

| 字段 | 内容 | 交互 |
|------|------|------|
| 运行状态 | ● 运行中 / ○ 已停止 | 点击展开启动/停止/重启按钮 |
| 代理模式 | 规则 / 全局 / 直连 | Segmented Picker 直接切换 |
| 监听地址 | 127.0.0.1:7890 | 只读展示 |
| Clash API | 127.0.0.1:9091 | 只读展示 |

- 状态指示器使用小圆点（绿色运行/红色停止）
- 操作按钮默认隐藏，点击状态区域或悬停时展开，避免误操作

### 2.2 卡片网格

2 列 `LazyVGrid` 布局，共 6 张卡片：

#### 第一行：实时数据

| 实时网速 | 活跃连接 |
|---------|---------|
| ↓ 下载速度（大数字） | 连接数（大数字） |
| ↑ 上传速度（大数字） | 进程数（小字） |
| 总下载量 / 总上传量（小字） | |

#### 第二行：流量统计

| 流量统计 | 流量趋势 |
|---------|---------|
| 今日 / 本月 切换（Segmented） | 轻量折线图 |
| 直连 XX MB / 代理 XX MB | 最近 1 小时，1 分钟粒度 |
| 水平条形图（直连/代理比例） | SwiftUI Charts `LineMark` |

#### 第三行：配置信息

| 配置目录 | 环境变量 |
|---------|---------|
| 路径展示 | `export https_proxy=...` |
| 「打开」按钮（Finder） | 「复制」按钮 |

### 2.3 卡片视觉规范

- 圆角：12pt
- 内边距：16pt
- 卡片间距：12pt
- 网格与状态条间距：16pt
- 背景：首选 `.ultraThinMaterial`，如有性能问题 fallback 到 `.quaternarySystemFill`
- 数据数字：`.system(.title, design: .rounded, weight: .semibold)`
- 标签文字：`.caption` + `.secondary` 前景色
- 单位文字：`.caption2` + `.tertiary` 前景色

## 三、流量趋势图规格

### 3.1 数据规格

- 粒度：1 分钟
- 保留时长：最近 1 小时（60 个数据点）
- 存储：纯内存，不持久化
- 数据结构：环形缓冲区（固定大小数组 + 写入指针）

### 3.2 图表实现

- 使用 SwiftUI Charts 的 `LineMark`
- 两条线：上传（橙色）、下载（绿色）
- Y 轴自适应，使用 `ByteCountFormatter` 格式化
- 禁用动画（`.chartAnimation(.disabled)`），避免每次数据更新触发重绘动画
- 不显示交互提示（tooltip），保持轻量

### 3.3 数据采集

- 每分钟从 Clash API 读取一次累计流量，计算差值作为该分钟的流量
- 采集逻辑独立于网速轮询（网速是 2 秒一次，流量趋势是 1 分钟一次）

## 四、性能约束

### 4.1 当前窗口生命周期（代码现状）

BoxX 的窗口关闭行为：

- **红色关闭按钮**：隐藏窗口，App 转为 accessory 模式（仅菜单栏），不退出
- **Cmd+Q**：`applicationShouldTerminate` 拦截，隐藏所有窗口，返回 `.terminateCancel`，不退出
- **真正退出**：仅通过菜单栏「退出 BoxX」触发（`shouldReallyQuit = true`）

当前问题：`StatusPoller.shared` 是单例，窗口关闭后定时器仍然持续运行（3-30 秒轮询），概览页的数据也保留在 `AppState` 单例中不释放。

### 4.2 窗口关闭零消耗（本次改造目标）

**原则：窗口不可见时，UI 侧的一切轮询和数据采集必须停止。**

需要引入窗口可见性状态（`windowVisible: Bool`），通过 `NSWindow.didBecomeMainNotification` / `onDisappear` 跟踪。

| 状态 | 网速轮询 | 流量统计 | 流量趋势采集 | StatusPoller |
|------|---------|---------|------------|-------------|
| 窗口可见 + 运行中 | 2 秒 | 30 秒 | 1 分钟 | 30 秒（保持现有） |
| 窗口可见 + 已停止 | 停止 | 停止 | 停止 | 3 秒（保持现有） |
| 窗口不可见 + 运行中 | 停止 | 停止 | 停止 | 停止 |
| 窗口不可见 + 已停止 | 停止 | 停止 | 停止 | 停止 |

窗口关闭时：
1. 停止概览页所有 `Timer` / `Task`（网速、流量统计、流量趋势）
2. 停止 `StatusPoller` 定时器
3. 释放流量趋势的历史数据数组
4. 清零网速、连接数等实时数据（下次打开重新拉取）
5. `AppState` 单例保留（配置、订阅等冷数据不释放，它们不消耗 CPU）
6. 仅保留菜单栏图标（接近零 CPU 消耗）

窗口重新打开时：
1. 重新启动 `StatusPoller`
2. 重新初始化流量趋势数据结构（空数组，重新开始采集）
3. 立即执行一次全量数据拉取（网速、连接数、流量等）

### 4.3 实现方案

在 `AppState` 中增加 `isWindowVisible` 属性：
- `MainView.onAppear` 设为 `true`
- `MainView.onDisappear` 设为 `false`
- 所有轮询逻辑检查此标志，`false` 时不调度下一次定时器
- `StatusPoller.start()` / `stop()` 由此标志驱动

注意：菜单栏的运行状态指示（图标颜色变化）不依赖 StatusPoller 的定时轮询，可通过 XPC 回调或进程存活检测实现，不需要持续轮询。

### 4.4 其他性能规则

- 全部使用 SwiftUI 原生组件，禁止 Canvas / 自定义绘制
- 卡片背景如果 `.material` 导致滚动卡顿，降级为纯色
- 流量趋势图数据点上限 60 个，超出时覆盖最旧数据
- sing-box 进程的运行不受 UI 状态影响

## 五、App 图标重设计

### 5.1 设计方向

以 sing-box 的等轴测包裹箱造型为基础，注入 BoxX 自己的视觉特色。

- **基础造型**：保留 sing-box 的等轴测（isometric）箱子轮廓，让用户一眼能联想到 sing-box
- **BoxX 特色**：在箱子上融入 BoxX 的品牌元素（如字母 X 的标记、开箱状态、或箱面纹理变化等）
- **色调**：浅色系背景，优先白色；箱体本身可用品牌色或柔和的对比色
- **风格**：macOS App 图标规范（圆角矩形），保持简洁现代感

### 5.2 约束

- 符合 macOS App Icon 尺寸规范（1024x1024 主尺寸）
- 在 16x16 到 512x512 各尺寸下都清晰可辨
- 与 sing-box 原始图标有明显区分，避免混淆
