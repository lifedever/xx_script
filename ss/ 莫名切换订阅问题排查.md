# Shadowrocket 莫名切换订阅问题排查

## 原因

配置文件中的 `update-url` 会让 Shadowrocket 定期自动拉取远程配置并重新应用，导致：

1. 代理组的手动选择被重置为默认值（`policy-select-name`、`select=0`）
2. 走的线路变了，看起来像"换了订阅"

## 解决方案

### 方案一：注释掉 update-url（推荐）

不需要远程自动更新时，注释掉 `shadowrocket.conf` 中的：

```conf
# update-url = https://fastly.jsdelivr.net/gh/lifedever/xx_script@main/ss/shadowrocket.conf
```

之后手动导入配置，不会被自动覆盖。

### 方案二：在 Shadowrocket 里关闭自动更新

设置 → 配置 → 关闭自动更新，只在需要时手动更新。

### 方案三：去掉默认选择参数

保留自动更新但不重置选择，移除配置中的 `select=0` 和 `policy-select-name=xxx` 参数，更新时不会强制覆盖手动选择。
